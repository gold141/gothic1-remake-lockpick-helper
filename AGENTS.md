# Gothic 1 Remake — Lockpick Helper v3

## Цель проекта

Интерактивный помощник для решения мини-игры взлома замков в игре **Gothic 1 Remake**.

В мини-игре **4–7 пластин** замка (настраивается). Каждая пластина содержит **7 позиций** (отверстий), пронумерованных сверху вниз от **1 до 7**. Штифт проходит через одно из отверстий. Задача — сдвинуть все штифты на центральную позицию **4**.

**Ключевая механика:** мы двигаем не штифт, а **сами пластины**. При движении пластины влево штифт относительно неё "переходит" на позицию вправо (и наоборот).

Проблема: пластины **связаны друг с другом**. При движении одной пластины некоторые другие тоже сдвигаются, иногда в противоположную (зеркальную) сторону. Связи разные для каждого нового замка.

Программа работает как **визуальный симулятор** — ты задаёшь начальное состояние пластин (как в игре), настраиваешь связи стрелками [↑][↓], программа решает головоломку через BFS, а затем **автоматически выполняет решение в игре** по нажатию F2.

## Механика мини-игры

| Параметр | Значение |
|----------|----------|
| Пластин | 4–7 (настраивается) |
| Позиций на пластине | 7 (1–7, сверху вниз) |
| Целевая позиция | 4 (центр) |
| Переключение пластин | Клик мышью в программе / W/S в игре |
| Движение пластины | A / D (влево/вправо) |
| Движение циклическое | **Нет** (упирается в край) |
| Связи пластин | 1:1 (связанные сдвигаются на 1 шаг, возможно зеркально) |
| Связи для замка | Меняются каждый раз при новом замке |

### Важно: двигаем пластины, а не штифт!

- Движение пластины **вправо** → штифт смещается влево (позиция уменьшается)
- Движение пластины **влево** → штифт смещается вправо (позиция увеличивается)

Это инвертирует математику модели.

## Возможности программы

### 1. Визуальный редактор
- **4–7 пластин** горизонтально (слева направо)
- Каждая пластина — вертикальный столбик из 7 кружков
- Красный круг = позиция штифта
- Нумерация **сверху вниз**: 1 — верх, 7 — низ

### 2. Режимы работы

| Кнопка | Режим | Описание |
|--------|-------|----------|
| **УСТАНОВИТЬ ПЛАСТИНЫ** | Редактор | Кликай по кружкам чтобы задать позицию штифта для каждой пластины |
| **НАСТРОИТЬ СВЯЗИ** | Редактор связей | 1. Кликни по пластине чтобы выбрать её как источник<br>2. Жми [↑] или [↓] у любой пластины чтобы задать связь |
| **СБРОС** | Сброс | Все пластины → центр (4), связи → единичная матрица |
| **РЕШИТЬ** | Решение | Запускает BFS, показывает пошаговую инструкцию |

### 3. Стрелки связей [↑] [↓]

Показывают, куда двигается пластина **i** при движении активной пластины **j вправо** (нажатие D):

| Стрелка | Цвет | Значение |
|---------|------|----------|
| **[↑]** | Зелёная | +1 — пластина двигается **в ту же сторону** (вправо) |
| **[↓]** | Красная | -1 — пластина двигается **зеркально** (влево) |
| **(серая)** | Серая | 0 — **нет связи** |

> **Диагональ** (связь пластины с самой собой) тоже настраивается! По умолчанию [↑] (+1), но можно переключить на [↓] (-1). Это значит, что при нажатии D пластина будет двигаться влево.

### 4. Автовыполнение (F2)

**Глобальная горячая клавиша F2** — работает даже когда программа НЕ в фокусе.

**При нажатии F2:**
1. **Подготовка**: нажимает **S × 8** (вниз к первой пластине)
2. **Выполнение**: поочерёдно выполняет все ходы из решения:
   - Выбирает нужную пластину (W/S)
   - Нажимает A или D
   - Пауза 0.5 сек между действиями
3. По завершении выводит сообщение в лог

**Повторное нажатие F2** — останавливает выполнение.

> ⚠️ **Требуется запуск от имени администратора** для работы глобальных горячих клавиш!

### 5. Мультиязычный интерфейс

- **4 языка**: Русский, English, Deutsch, Español
- Переключатель в верхней панели
- Все строки интерфейса, логов и сообщений переведены

### 6. Выбор количества пластин

- Выпадающий список: **4, 5, 6, 7**
- При изменении → **автоматический сброс**
- Окно программы автоматически подстраивает ширину

## Алгоритм решения (BFS)

Когда заданы позиции и связи:

- **Состояние**: кортеж позиций штифтов `(p1, p2, ..., pn)`, каждая 1–7
- **Цель**: `(4, 4, ..., 4)`
- **Переходы**: `2 × n` действий (n пластин × влево/вправо)
- **Поиск**: BFS по пространству состояний (максимум `7^n`)
- **Ограничения**: проверка границ 1–7 (движение не циклическое)
- **Вывод**: кратчайшая последовательность с группировкой одинаковых ходов

**Формат вывода:**
```
✅ Решение: 5 ходов
3→ 2← ← 1→ 4←
```
> Одинаковые пластины подряд группируются: `2← ←` вместо `2← 2←`
> Стрелка показывает физическое направление **движения пластины** (→ = вправо, ← = влево)

## Математическая модель

Пусть `C` — матрица связей размером `n×n` (n = 4–7).

- `C[i][j]` — на сколько сдвигается **пластина** `i` (0-based), когда пластина `j` двигается **вправо** на 1 шаг (нажатие D).
- Диагональ: `C[j][j]` = +1 (двигается вправо) или -1 (двигается влево при нажатии D)
- Возможные значения: `-1` (зеркально влево), `0` (нет связи), `1` (в ту же сторону вправо)

**Формула обновления позиции штифта** при нажатии D (`direction = +1`) или A (`direction = -1`) для пластины `j`:
```
new_pin[i] = old_pin[i] - direction * C[i][j]
```

> Минус перед `direction` потому что при движении пластины вправо штифт смещается влево!

**Проверка границ:** `1 <= new_pin[i] <= 7` для всех `i`. Если нарушение — ход невозможен.
